# POS Constitution

> The highest authority governing this system. All code, features, and decisions must comply.

---

## I. Purpose

This system exists to:
1. **Track my evolution** — daily life, learning, thinking, failures, and growth
2. **Preserve truth** — honest documentation without distortion or revisionism
3. **Prove effort** — undeniable evidence of work, struggle, and progress
4. **Enable reflection** — allow future-me to study past-me objectively

This is NOT:
- A portfolio or social showcase
- A productivity tool optimized for output
- A place to impress anyone

---

## II. What This System Must NEVER Become

1. **Never a performance** — No writing for an audience. Write for yourself at 50.
2. **Never sanitized** — Failures, doubts, and embarrassments stay recorded.
3. **Never gamified** — Metrics serve insight, not dopamine.
4. **Never abandoned** — Gaps are documented, not hidden.
5. **Never over-engineered** — Simplicity ensures longevity.

---

## III. Sacred Data

The following data categories are **inviolable**:

| Category | Rule |
|----------|------|
| Journal entries | Once locked, NEVER modified. Add addendums only. |
| Timestamps | NEVER backdated or altered. |
| Failures | NEVER deleted. Can be contextualized, not erased. |
| Decisions | NEVER removed. Reasoning preserved forever. |
| Git history | NEVER rewritten (no force-push, no rebase on vault). |

---

## IV. Rules for Editing vs Archiving

### Editable (Before Lock)
- Journal entries within 48 hours of creation
- Draft notes not yet marked "published"
- Project documentation during active phase

### Immutable (After Lock)
- Journal entries after 48-hour lock
- Archived project outcomes
- Decision records
- Achievement proofs

### Addendum Allowed
- You may ADD a "Reflection" section to locked entries
- You may ADD context to old decisions
- You may NEVER modify the original text

---

## V. Rules for AI

1. **AI is read-only** — AI may analyze, summarize, and suggest. It may NEVER:
   - Modify files directly
   - Auto-generate entries attributed to you
   - Overwrite any human-written content

2. **AI outputs are marked** — Any AI-generated content must be clearly labeled:
   ```
   <!-- AI-GENERATED: 2026-01-18 -->
   ```

3. **AI is optional** — The system must function fully without AI.

4. **AI is disposable** — All AI-derived data (embeddings, reflections) can be deleted and regenerated.

---

## VI. Deletion Policy

**When future-you is allowed to delete data:**

| Allowed | NOT Allowed |
|---------|-------------|
| AI-generated caches | Journal entries |
| Temporary/draft files | Locked reflections |
| Duplicate media | Decision records |
| Test data | Any data after lock |

**The 10-Year Rule**: If you want to delete something meaningful, wait 10 years. If it still feels right, add a deletion record explaining why — then delete.

---

## VII. Amendment Process

This constitution can be amended, but:
1. Changes require a Decision Log entry explaining the reasoning
2. Previous version must be preserved in Git history
3. Amendments cannot retroactively justify past violations
